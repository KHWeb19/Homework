[메서드 작동 원리 이해]
:어떤 행동을 표현하는 방법

1단계 메서드 기본 틀 만들기 

EX)핫케이크 만들기

핫케이크만들기{
	준비물을 준비합니다;
	준비물을 그릇에 넣고 잘 섞어 빵 반죽을 만듭니다;
	빵 반죽을 프라이팬에 넣고 1분 정도 가열해 줍니다;
	구워진 빵 위에 과일을 올립니다;
}

2단계 외부 값 가져오기

메서드에도 외부에서 값을 가지고 올 수 있는 창구를 만들어놓았습니다.

핫케이크만들기(double 버터, double 우유, double 달걀, double 과일) {
	버터, 우유, 달걀을 그릇에 넣고 잘 섞어 빵 반죽을 만듭니다;
	빵 반죽을 프라이팬에 넣고 1분 정도 가열해 줍니다;
	구워진 빵 위에 과일을 올립니다;
}

메서드 이름 다음에 괄호() 공간을 만들어서 외부에서 전달받을 변수들의 변수형과 이름을 선언하면 됩니다. 이후 메서드 안에서 조리법을 설명할 때에도 맨 앞에서 선언한 변수형을 그대로 사용하면 됩니다.

3단계 메서드의 결과물 돌려주기(리턴)

만들어진 핫케이크를 외부에 전달할 방법

int 핫케이크만들기(double 버터, double 우유, double 달걀, double 과일) {
	버터, 우유, 달걀을 그릇에 넣고 잘 섞어 빵 반죽을 만듭니다;
	빵 반죽을 프라이팬에 넣고 1분 정도 가열해 줍니다;
	구워진 빵 위에 과일을 올립니다;	
	return 만들어진_핫케이크_개수;
}

메서드에서 만든 값을 외부에 전달하는 전달자는 리턴return입니다. 메서드 이름앞에 전달될 변수형을 선언하고, 메서드 안에 (일반적으로 마지막에) 메서드에서 열심히 만든 결과물을 전달하는 리턴 명령을 사용하면 간단히 표현됩니다.

4단계 결과를 돌려줄 필요가 없을 때 사용하는 빈 공간

만약에 리턴값이 필요 없는 경우라면 필요 없다는 표시를 해주어야 합니다. 이때 사용되는 단어가 void입니다.

리턴값이 없는 메서드는 void를 메서드 이름 앞에 적어주고, 메서드 내용에 return을 적지 않습니다.

void 핫케이크만들기(double 버터, double 우유, double 달걀, double 과일) {
	버터, 우유, 달걀을 그릇에 넣고 잘 섞어 빵 반죽을 만듭니다;
	빵 반죽을 프라이팬에 넣고 1분 정도 가열해 줍니다;
	구워진 빵 위에 과일을 올립니다;	
	핫케이크를 맛있게 먹습니다;
}

메인 메서드와 보조 메서드

메서드를 한번 정의하고 나면, 필요할 때마다 불러와서 사용할 수 있습니다. 이것을 '메서드를 호출한다'고 말합니다. 호출은 어떻게 할까요? 메서드의 이름을 부르면 됩니다.

메인(){
	단팥빵만들기();
	식빵만들기();
	단팥빵만들기();
}

}
